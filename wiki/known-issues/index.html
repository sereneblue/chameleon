URLs exempt from Chameleon spoofing

https://accounts.google.com

Chameleon does not spoof anything when accessing a URL that starts with accounts.google.com. More details can be found here.

https://accounts.youtube.com

Chameleon does not spoof anything when accessing a URL that starts with accounts.youtube.com. More details can be found here.

https://disqus.com/embed/comments/

Chameleon does not spoof the iframe when accessing embedded Disqus comments.

https://www.google.com/recaptcha/api2

Recaptcha is already inconvenient with regular Firefox, it becomes almost impossible when Chameleon is enabled.

https://docs.google.com/

Timezone spoofing breaks Google Docs. Non-Firefox profiles may also cause issues.

https://drive.google.com/

Timezone spoofing breaks Google Drive. Non-Firefox profiles may also cause issues.

URLs known to have issues. You should whitelist them.

https://open.spotify.com

Some profiles do not work with Spotify (IE11).

https://twitter.com

Twitter is known to have issues ("Something went wrong"). Using a non-Firefox user agent can cause issues.

https://web.whatsapp.com

WhatsApp checks for some features available in the browser depending on the user agent. This causes issues with some profiles.
